## Engineering posture

I separate architecture claims from implementation evidence.

A system is not considered mature solely because it has an interface or a compelling concept. I look for:

1. source in the repository;
2. runnable commands and defined boundaries;
3. tests, benchmarks, or deterministic validation;
4. deployment or packaging paths;
5. operator documentation;
6. receipts, hashes, manifests, and auditability;
7. explicit disclosure of missing credentials, external infrastructure, signing, notarization, monitoring, or independent audit.

I distinguish simulation from live execution, local validation from external certification, and agent intelligence from owner authority.
