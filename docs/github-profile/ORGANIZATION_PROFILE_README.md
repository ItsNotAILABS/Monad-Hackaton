## What we are building

ItsNotAI Labs is an independent research and engineering lab developing a connected architecture for:

- persistent and stateful AI systems;
- governed local and cloud execution;
- agent identity, credentials, memory, and receipts;
- scientific intelligence and spectral computation;
- AI-native financial research and market infrastructure;
- blockchain application creation and policy-controlled wallet interaction;
- MCP-based federation across tools, models, runtimes, devices, and applications;
- deployable capsules, canisters, desktop applications, edge services, and research artifacts.

The work is organized as interoperable system families rather than isolated demos. Public repositories expose selected implementations, research programs, SDKs, protocols, product surfaces, and proof artifacts. Private repositories preserve unreleased runtimes, operator infrastructure, security-sensitive components, and active experimental systems.

> **Agents propose. Policy evaluates. Owners approve. Wallets sign. Receipts remember.**

### Synthetic cognition and entity architecture

Our entity research treats intelligence as a recurrent brain-body control system with canonical state, memory, consequence, adaptation, identity, and longitudinal evaluation.

Core architectural modules include:

`StateManager` · `PerceptionEngine` · `SalienceEngine` · `WorkingMemoryGate` · `ArbitrationEngine` · `ActionSelector` · `ConsequenceEngine` · `HomeostasisEngine` · `MemoryEngine` · `RecurrenceEngine` · `AdaptationEngine` · `DriftDetector` · `IdentityEngine` · `BenchmarkHarness`

## Proof-first engineering

A capability is not treated as mature because it has a compelling name or interface. Production claims should be supported by evidence such as:

1. source present in the repository;
2. a runnable command or reachable interface;
3. validation, tests, benchmarks, or deterministic checks;
4. a deployment target or packaging path;
5. operator documentation and explicit boundaries;
6. receipts, hashes, manifests, or release evidence;
7. honest disclosure of external credentials, infrastructure, signing, custody, monitoring, or audit requirements.

We distinguish:

- **implemented source** from proposed roadmap;
- **local validation** from external audit;
- **simulation and planning** from live financial execution;
- **package generation** from signed and notarized distribution;
- **configured deployment** from verified public availability;
- **agent proposals** from owner-approved action.

## Governance and safety boundary

ItsNotAI Labs designs agents and runtimes to operate under explicit authority:

```text
Observation
  → proposal
  → policy evaluation
  → owner or delegated approval
  → external signing / governed execution
  → result collection
  → receipt and audit record
```

Sensitive capabilities should be narrow, revocable, namespaced, observable, and receipt-bearing. Private keys, seed phrases, unrestricted wallet authority, hidden command execution, and silent model substitution are outside the default architecture.
